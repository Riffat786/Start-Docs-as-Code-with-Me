# Root Files

## package.json

The `package.json` file is the central configuration file for a Node.js project.

It contains:

- Project metadata
- Dependencies
- Build scripts
- Version information

Example:

```json
{
  "name": "start-docs-as-code-with-me",
  "private": true
}
```

Common scripts include:

| Script | Purpose |
|---------|---------|
| `npm start` | Start the development server |
| `npm run build` | Create a production build |
| `npm run serve` | Preview the production build |
| `npm run deploy` | Deploy the website |

---

## package-lock.json

The `package-lock.json` file records the exact versions of installed dependencies.

It ensures that everyone working on the project installs identical package versions.

> **Best Practice:** Do not edit this file manually.

---

## docusaurus.config.ts

This is the main configuration file for your website.

It controls:

- Website title
- Tagline
- URL
- Navigation bar
- Footer
- Theme
- Plugins
- Deployment settings

This file is covered in detail in the next lesson.

---

## sidebars.ts

This file controls the documentation sidebar.

You can:

- Organize topics
- Group documentation
- Create categories
- Change navigation order

---

## tsconfig.json

This file configures TypeScript.

Most projects can use the default configuration unless custom TypeScript settings are required.

---

## README.md

The repository homepage displayed on GitHub.

This file introduces the project, explains its purpose, and provides setup instructions.

---

## .gitignore

The `.gitignore` file specifies files and folders Git should ignore.

Typical entries include:

```text
node_modules/
build/
.cache/
```

# Best Practices

- Keep configuration files in the project root.
- Store documentation inside the `docs` folder.
- Store images inside the `static` folder.
- Use descriptive folder names.
- Keep the project organized as it grows.
