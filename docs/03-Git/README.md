# 🌿 Module 4 · Git

Git is the world's most widely used version control system and is at the heart of every Docs-as-Code workflow. It allows documentation teams to track changes, collaborate efficiently, and maintain a complete history of every document.

In this module, you'll learn the essential Git concepts and commands used in real-world documentation projects.

---

## 🎯 Learning Objectives

After completing this module, you will be able to:

- Understand Git repositories.
- Track documentation changes.
- Work with branches.
- Merge changes safely.
- Recover previous versions.
- Use Git confidently in daily workflows.

---

## 📚 Topics Covered

| Topic | Description |
|--------|-------------|
| Repository | Create and manage repositories. |
| Commits | Save documentation changes. |
| Branches | Work on features independently. |
| Merge | Combine changes from multiple branches. |
| Rebase | Keep commit history clean. |
| Tags | Mark important releases. |
| Clone | Copy repositories locally. |
| Stash | Save temporary work. |
| Undo | Recover from mistakes. |
| Cheat Sheet | Quick reference for Git commands. |
